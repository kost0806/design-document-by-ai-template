# 요구사항 분석 AI 프롬프트 템플릿

이 템플릿을 사용하여 AI에게 요구사항 분석을 요청할 수 있습니다.

---

## 📋 프롬프트 템플릿

```
당신은 소프트웨어 요구사항 분석 전문가입니다. 다음 프로젝트의 요구사항을 체계적으로 분석하고 문서화해주세요.

### 프로젝트 개요
- **프로젝트명**: [프로젝트 이름]
- **목적**: [프로젝트의 목적과 해결하려는 문제]
- **대상 사용자**: [주요 사용자 그룹]
- **예상 규모**: [예상 사용자 수, 데이터 규모 등]

### 핵심 기능 (예시)
1. [기능 1 설명]
2. [기능 2 설명]
3. [기능 3 설명]

### 제약사항
- **기술적 제약**: [사용 가능한 기술 스택, 인프라 제한 등]
- **비즈니스 제약**: [예산, 일정 등]
- **법적/규제 제약**: [개인정보보호법, 보안 요구사항 등]

### 요청사항
다음 형식으로 요구사항을 작성해주세요:

1. **기능 요구사항**
   - 사용자 스토리 형식으로 작성
   - MoSCoW 우선순위 표시
   - 각 기능의 입력/처리/출력 명시

2. **비기능 요구사항**
   - 성능 요구사항 (응답 시간, 처리량)
   - 보안 요구사항
   - 확장성 요구사항

3. **제약사항**
   - 기술적/비즈니스/법적 제약사항 정리
```

---

## 💡 사용 예시

### 입력 (프로젝트 정보)
```
프로젝트명: 온라인 도서 대여 플랫폼
목적: 사용자가 온라인으로 도서를 검색하고 대여할 수 있는 플랫폼
대상 사용자: 일반 독서가, 학생
예상 규모: 10,000명

핵심 기능:
1. 도서 검색 및 필터링
2. 도서 대여 및 반납
3. 사용자 리뷰 작성
```

### 출력 (AI 생성 결과 예시)
AI가 기능 요구사항, 비기능 요구사항, 제약사항을 체계적으로 작성해줍니다.

---

## ✅ 체크리스트

프롬프트 사용 전 확인사항:
- [ ] 프로젝트 개요가 명확한가?
- [ ] 핵심 기능이 나열되어 있는가?
- [ ] 제약사항이 파악되어 있는가?
- [ ] 대상 사용자가 정의되어 있는가?

---

## 🔄 반복 개선 팁

1. **1차 프롬프트**: 위 템플릿으로 초안 생성
2. **2차 프롬프트**: "FR-001의 예외 처리를 더 상세하게 작성해주세요"
3. **3차 프롬프트**: "성능 요구사항에 구체적인 수치를 추가해주세요"

AI와 대화하며 점진적으로 문서를 개선하세요!
