# Detailed Level Design (DLD)
{: .no_toc }

각 컴포넌트의 상세 구현 사항을 설계합니다.
{: .fs-6 .fw-300 }

---

## 개요

DLD는 HLD에서 정의한 아키텍처와 컴포넌트를 바탕으로, 실제 구현에 필요한 상세 설계를 다룹니다.

### 🎯 목적

- API 상세 명세
- 데이터 모델 설계
- 클래스/모듈 구조
- 알고리즘 및 로직 정의

---

## 📑 문서 구성

### [API 명세 (API Specification)](api-specification/)
RESTful API의 상세 명세를 작성합니다.

### [데이터 모델 (Data Model)](data-model/)
데이터베이스 스키마와 엔티티 관계를 정의합니다.

### [클래스 설계 (Class Design)](class-design/)
주요 클래스와 모듈의 구조를 정의합니다.

### [구현 상세 (Implementation Details)](implementation-details/)
